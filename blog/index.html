[Akka gRPC] Getting Started

2021년 11월 4일 · 약 3분
Jay Kim
책임 연구원 / ELFiNOS팀

Akka gRPC Plugin을 이용해 gRPC Sample Code를 만들어 본다.

예제에서 사용한 버전 정보#

  • JDK 11
  • Scala 2.13.7
  • sbt 1.3.9

프로젝트 생성#

info

본 code는 IntelliJ 기준에서 제작하였습니다.

File -> New -> Project 에서 sbt 프로젝트를 생성한다.

Plugin, Library 설정#

project 디렉토리 안에 plugins.sbt 파일을 생성 후 아래 코드를 입력한다.

addSbtPlugin("com.lightbend.akka.grpc" % "sbt-akka-grpc" % "2.1.0")

build.sbt 파일에 아래 코드를 입력한다.

enablePlugins(AkkaGrpcPlugin)

.proto 파일 생성#

src/main/protobuf 에 test.proto 파일을 생성 후 아래 코드를 입력한다.

syntax = "proto3";
package protobuf;
message MachineRequest {    string ncId = 1;}
message MachineResponse {    string message = 1;}
service ElfinService {  rpc GetMachineInfo(MachineRequest) returns (MachineResponse) {}}

terminal을 켜서 아래 커맨드를 입력하면 관련 코드가 자동 생성 된다.

sbt compile

기능 구현#

src/main/scala/ElfinServiceIm 파일을 생성 후 아래 코드를 입력한다.

class ElfinServiceIm extends ElfinService {  override def getMachineInfo (in : MachineRequest) : Future[MachineResponse] = {    Future.successful(MachineResponse(s"NC${in.ncId} is RUNNING"))  }}

Akka Http Server와 연결#

src/main/scala/ElfinService 파일을 생성 후 아래 코드를 입력한다.

import akka.actor.typed.ActorSystemimport akka.actor.typed.scaladsl.Behaviorsimport akka.http.scaladsl.Httpimport akka.http.scaladsl.model.{HttpRequest, HttpResponse}import com.typesafe.config.ConfigFactoryimport protobuf.ElfinServiceHandler
import scala.concurrent.{ExecutionContext, Future}
object ElfinServer extends App{  val config = ConfigFactory    .parseString("akka.http.server.preview.enable-http2 = on")    .withFallback(ConfigFactory.defaultApplication())
  val system = ActorSystem[Nothing](Behaviors.empty, "elfinos_api", config)
  new ElfinServer(system).run}
class ElfinServer(system: ActorSystem[_]){  def run : Future[Http.ServerBinding] = {    implicit val sys = system    implicit val ec : ExecutionContext = system.executionContext
    val service : HttpRequest => Future[HttpResponse] =      ElfinServiceHandler(new ElfinServiceIm)
    val bound : Future[Http.ServerBinding] = Http(system)      .newServerAt("localhost", 8080)      .bind(service)
    bound.foreach{      binding => println(s"gRPC server bound to : ${binding.localAddress}")    }    bound  }}

[AkkaServerless] Getting Started

2021년 10월 20일 · 약 2분
Jay Kim
책임 연구원 / ELFiNOS팀

Giter8 을 이용해 lightbend에서 제공하는 템플릿을 사용하면 빠르게 개발이 가능하다.

Prerequisites#

  • Java 11 이상
  • Docker 20.10.18 이상
  • sbt 1.3.6 이상

Generate and build the Akka Serverless project#

  1. Teminal을 열고 아래 커맨드를 입력한다.
 sbt new lightbend/akkaserverless-value-entity.g8

  1. 새롭게 만들어진 프로젝트로 안으로 들어간다.

  2. sbt compile 명령으로 소스를 컴파일하면 .proto 파일을 기반으로 개발에 필요한 class, method 등이 자동으로 만들어진다.

Package service#

이 프로젝트는 Aka Serverless에 배포할 수 있는 Docker 이미지로 서비스를 패키징하도록 구성되어 있다.

아래 커맨드를 이용해 repositoy에 도커 아미지를 배포한다.

sbt -Ddocker.username=alice docker:publish

Run locally#

로컬 환경에서 어플리케이션을 실행하기 위해 akkaserverless proxy를 시작해야 한다. 프로젝트내 docker-compose 파일에 proxy를 실행하기 위해 필요한 부분이 설정되어 있다.

proxy를 시작하기 위해 아래 커맨드를 프로젝트 디렉토리에서 실행한다.

docker compose up

어플리케이션을 시작하기 위해 아래 커맨드를 프로젝트 디렉토리에서 실행한다.

sbt run

[Cassandra] field larger than field limit (131072)

2021년 10월 1일 · 약 1분
Jay Kim
책임 연구원 / ELFiNOS팀

현상#

CQL 에서 .csv 파일 copy시 오류 메시지 출력 field larger than field limit (131072)

원인#

  • CQL 에서 .csv 파일 copy 시 field의 값이 제한을 넘어 문제가 생김

해결 방안#

  1. $HOME 디렉토리 아래에 .cqlshrc 파일을 찾는다
  2. 아래의 구문을 추가한다
  [csv]  field_size_limit = 1000000000
  1. field_size_limit의 값은 상황에 맞게 조절한다.

[PlayFramework] Window 에서 명령줄이 길다고 실행 되지 않을 때

2021년 10월 1일 · 약 1분
Jay Kim
책임 연구원 / ELFiNOS팀

현상#

Window 환경에서 CMD로 서버를 실행 할때 명령줄이 길어서 실행이 안 되는 경우가 있음.

원인#

  • 명령 프롬프트에서 다음 명령줄의 총 길이는 8191자 이상을 포함할 수 없다.

해결 방안#

  • 프로젝트/bin/프로젝트.bat를 메모장으로 연다.

  • set "APP_CLASSPATH=%APP_LIB_DIR%..\conf\;%APP_LIB_DIR%......" 을 다음과 같이 수정한다

    set "APP_CLASSPATH=%APP_LIB_DIR%\..\conf\;%APP_LIB_DIR%\*"
